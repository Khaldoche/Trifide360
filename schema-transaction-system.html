7. Gestion des Transactions et Historique

Vue d'Ensemble du Système Transactionnel

Le système de gestion des transactions de Trifide 360 assure l'intégrité des données géospatiales lors des opérations d'édition complexes. Il combine transactions ACID, historique complet des modifications, mécanismes d'annulation/rétablissement (Undo/Redo), et gestion des conflits pour maintenir la cohérence des données cartographiques.

Architecture Transactionnelle

Pipeline Transactionnel

Début Transaction ↓ Validation Prérequis ↓ Exécution Operations (Batch) ↓ Validation Postconditions ↓ Commit/Rollback Decision ↓ Journal Historique ↓ Notification Changements

Types de Transactions

Transactions Simples

  • Édition feature - Modification géométrie/attributs
  • Création objet - Nouveau point/ligne/polygone
  • Suppression - Retrait élément unique
  • Déplacement - Translation géométrique

Transactions Complexes

  • Import fichier - Chargement données massives
  • Opérations géométriques - Buffer, intersection, union
  • Projections - Transformation systèmes coordonnées
  • Édition batch - Modification multiple features

Transactions Distribuées

  • Multi-couches - Modifications plusieurs datasets
  • Inter-modules - WPF ↔ Unity synchronisation
  • Sauvegarde projet - État complet application
  • Synchronisation données - Mise à jour externe

Exemples d'Utilisation

Transaction Simple

// Modification coordonnées point avec transaction using (var transaction = TransactionManager.BeginTransaction("Move Point")) { try { var originalGeometry = feature.Geometry.Clone(); // Modification coordonnées feature.Geometry = new Point(newX, newY, newZ); // Validation nouvelle position if (!GeometryValidator.IsValid(feature.Geometry)) { transaction.Rollback(); return false; } // Commit transaction transaction.Commit(); // Ajout à l'historique Undo UndoRedoManager.AddCommand(new MovePointCommand(feature, originalGeometry)); return true; } catch (Exception ex) { transaction.Rollback(); Logger.Error($"Transaction failed: {ex.Message}"); return false; } }

Transaction Complexe avec Batch

// Import fichier avec validation et rollback automatique public async Task<ImportResult> ImportShapefileWithTransaction(string filePath) { using (var transaction = TransactionManager.BeginTransaction("Import Shapefile")) { var importedFeatures = new List<Feature>(); try { var features = ShapeFileReader.ReadFeatures(filePath); foreach (var feature in features) { // Validation avant ajout var validationResult = FeatureValidator.Validate(feature); if (!validationResult.IsValid) { Logger.Warning($"Invalid feature {feature.ObjectID}: {validationResult.ErrorMessage}"); continue; } // Ajout à la couche await layerData.AddFeatureAsync(feature); importedFeatures.Add(feature); } // Commit si tout s'est bien passé transaction.Commit(); // Commande Undo pour l'import complet UndoRedoManager.AddCommand(new ImportCommand(importedFeatures, layerData)); return new ImportResult { Success = true, ImportedCount = importedFeatures.Count }; } catch (Exception ex) { // Rollback automatique en cas d'erreur transaction.Rollback(); return new ImportResult { Success = false, ErrorMessage = ex.Message }; } } }
